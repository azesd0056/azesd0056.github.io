博客搭建笔记

2022/12/28

利用hexo + github page生成个人博客的方法过程，记录在博客搭建过程中遇到的问题和解决方法。一站式保姆级教程～

1.安装Node.js

下载地址：nodejs.org
下载 18.12.1 LTS 版本
安装成功后，终端切换到root用户：
sudu su
查看node/npm版本：node -v / npm-v

2.安装Hexo博客框架

借助npm包管理器安装 ，由于国内镜像源速度较慢，利用npm安装cnpm（淘宝镜像源）进行安装：
npm install -g cnpm –registry=https://registry.npm.taobao.org
安装完成后查看cnpm版本：
cnpm -v

通过cnpm安装Hexo：
cnpm install -g hexo-cli
安装完成后查看Hexo版本：
hexo -v

3.利用hexo搭建博客

新建一个blog文件夹用于存放搭建blog所需的所有文件（/Users/sd./blog）
在终端中cd到blog目录，初始化博客：
sudu hexo init
初始化后blog目录下生成了初始化的文件，查看：
ls -l

启动博客：
hexo s
在本地浏览器输入地址进行查看：
http://localhost:4000/

新建一篇博文：
首先退出hexo服务：Ctrl+C
hexo n “My First Post”
完成后在/Users/sd./blog/source/_posts/目录下生成一个My-First-Post.md文件
cd到该目录下，通过vim对My-First-Post.md文件内容进行修改

修改完成后，cd回blog目录，对hexo进行清理：
hexo clean
生成静态文件：
hexo g
重新启动hexo服务：
hexo s

4.将博客部署到远程站点

将博客部署到github上公开使用
首先断开hexo服务

登陆github账户，新建一个新的仓库：
new repository，仓库名命名为： 用户名+.github.io （aze0056.github.io）

回到终端，在blog目录下安装git部署的插件：
cnpm install –save hexo-deployer-git

安装完成后，对_config.yml文件进行设置：
vim _config.yml
文件最底部更改type，并添加两行：
type: git
repo: https://github.com/aze0056/aze0056.github.io.git （刚才创建的github仓库的网址）
branch: master
保存退出

部署到远端：
hexo d
